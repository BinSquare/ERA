## What is ERA Agent?

ERA Agent is a **secure code execution platform** that runs your code in isolated Firecracker microVMs. It supports multiple languages and provides session persistence for stateful workflows.

### Key Features

- **🔒 Secure Isolation** - Every execution runs in an isolated VM
- **💾 Session Persistence** - Maintain state across multiple executions
- **⚡ Multiple Languages** - Python, Node.js, TypeScript, Go, Deno
- **🌍 Deploy Anywhere** - Cloud (Cloudflare) or local (self-hosted)
- **📝 Multiple Interfaces** - REST API, CLI, MCP (Model Context Protocol)
- **📦 File Operations** - Upload, read, and manage files in sessions

### Use Cases

- **AI-Powered Tools** - Execute code generated by AI assistants like Claude
- **Code Playgrounds** - Build interactive code execution environments
- **CI/CD Pipelines** - Run tests and build scripts in isolated environments
- **Data Processing** - Process data with persistent sessions
- **Educational Platforms** - Safe code execution for learning
- **API Integrations** - Execute code in response to API calls

## Quick Examples

### Python Execution
```bash
curl -X POST https://era-agent.yawnxyz.workers.dev/api/execute \
  -H "Content-Type: application/json" \
  -d '{
    "language": "python",
    "code": "print(\"Hello World!\")\nprint([x**2 for x in range(10)])"
  }'
```

### Persistent Sessions
```bash
# Create session
curl -X POST https://era-agent.yawnxyz.workers.dev/api/sessions \
  -d '{"language":"python","persistent":true}'

# Run code (state persists)
curl -X POST https://era-agent.yawnxyz.workers.dev/api/sessions/{id}/run \
  -d '{"code":"x = 42"}'

curl -X POST https://era-agent.yawnxyz.workers.dev/api/sessions/{id}/run \
  -d '{"code":"print(f\"x = {x}\")"}'  # x is still available!
```

### Claude Desktop Integration
Add to your Claude Desktop config:
```json
{
  "mcpServers": {
    "era-agent": {
      "url": "https://era-agent.yawnxyz.workers.dev/mcp/v1"
    }
  }
}
```

### Local CLI Usage
```bash
# Build the agent
go build -o agent

# Create a VM session
./agent vm create --language python

# Run code
./agent vm run --vm <id> --cmd "python -c 'print(\"Hello!\")'"

# Start MCP server for Claude
./agent mcp

# Start HTTP server
./agent serve
```

## Deployment Options Comparison

| Feature | Hosted (Cloudflare) | Local (Self-Hosted) |
|---------|-------------------|---------------------|
| **Setup** | Zero config | Build from source |
| **Infrastructure** | Managed | Docker/Firecracker |
| **Scaling** | Automatic | Manual |
| **Network Access** | Enabled by default | Disabled by default |
| **State Storage** | Durable Objects (cloud) | BoltDB (local) |
| **File Storage** | R2 Storage (cloud) | Local filesystem |
| **Latency** | 50-200ms (global edge) | ~10ms (local) |
| **Privacy** | Data goes to Cloudflare | Everything stays local |
| **Cost** | Pay per use | Free (your hardware) |
| **Best For** | Production, AI tools | Development, sensitive data |
